Today Widget实战

发表于 2017-05-14 |

什么是Today Widget

today widget是App Extensions中的一种，可以让用户快速的访问重要的信息，例如：查看天气，获取股票信息，查看清单列表等等。就算在锁屏状态下用户依旧可以使用widget。（以下简称widget）

Apple 指出，确保widget用于提供你认为最合适的任务，可以帮助用户快速更新数据，或者完成简单的任务，不要将一些冗长的任务和功能加到widget中。

widget的使用建议：

  • 确保内容看起来总是最新的
  • 对于用户交互的反馈能够表现良好
  • 需要注意内存占用的限制，否则系统将会终止widget

    创建一个Today Widget

    因为扩展是一个独立的进程，所以我们需要新建一个target。

现在我们的工程中多出了一个和target同名的文件夹，MainInterface的storyboard就是我们的程序入口，对应是TodayViewController的ViewController。我们可以直接在storyboard中布局我们想要的UI，然后再ViewController中做逻辑处理。

当然如果你希望使用纯代码来完成UI，可以修改Info.plist中的NSExtension字段

  • 删除NSExtensionMainStoryboard字段，并且删除MainInterface的storyboard
  • 新增一个NSExtensionPrincipalClass字段，并设为TodayViewController（你也可以指定其他的ViewController）

不管是使用storyboard还是使用纯代码，我们只要像平时在正常app里写代码一样，在ViewController中进行处理。

UI

UI实现部分的就不说了，只说一些注意点和坑。
由于需要兼容iOS 10和iOS 10之前的版本，所以我们需要两套UI配色，根据系统版本去切换不同的配色，在viewDidLoad方法中进行处理即可。

  if (IS_DEVICE_RUNNING_IOS_10_AND_ABOVE) {
    textColor = kTextLightGrayColor;
    lineColor = [UIColor colorWithWhite:0 alpha:0.3];
    borderColor = [UIColor colorWithWhite:0 alpha:0.5];
}else {
    textColor = kTextWhiteColor;
    lineColor = [UIColor colorWithWhite:1 alpha:0.8];
    borderColor = [UIColor colorWithRed:221/255.0 green:221/255.0 blue:221/255.0 alpha:0.3];
    self.preferredContentSize = CGSizeMake(0, 170);
}

iOS 10之前

我的做法是iOS 10之前直接使用self.preferredContentSize = CGSizeMake(0, 170);来设置widget展示视图的大小。当然你需要动态的改变大小的话，可以自己在对应的事件里去改变preferredContentSize。

我们可以看到这时候widget是处于偏移的状态，这时候我们可以使用NCWidgetProviding给出的协议方法来解决。

- (UIEdgeInsets)widgetMarginInsetsForProposedMarginInsets:(UIEdgeInsets)defaultMarginInsets{
return UIEdgeInsetsMake(0, 10, 0, 10);
}

iOS 10上折叠和展开处理

iOS 10之后NSExtensionContext新增了widgetLargestAvailableDisplayMode属性，用来确认当前widget是展开还是折叠状态。折叠状态的值是固定的大小，展开状态的时候大小可以自定义，如果我们只需要一个固定的大小的话，就使用和iOS 10之前一样的写法就行。如果我们希望widget可以在折叠和展开的时候显示不同的UI，我们可以在DisplayMode属性值改变的时候去改变widget的高度。

如果我们希望widget初始化状态就是展开，我们可以在viewWillAppear中将widget的model设为NCWidgetDisplayModeExpanded。记得加上版本判断。

- (void)viewWillAppear:(BOOL)animated {
[super viewWillAppear:animated];
if (IS_DEVICE_RUNNING_IOS_10_AND_ABOVE) {
    self.extensionContext.widgetLargestAvailableDisplayMode = NCWidgetDisplayModeExpanded;
  }
}

当我们将widgetLargestAvailableDisplayMode的值设为NCWidgetDisplayModeExpanded，3DTouch上会自动加上widget。

当我们改变widgetLargestAvailableDisplayMode的值时系统会调用widgetActiveDisplayModeDidChange方法。我们可以在这个方法中处理高度。

// If implemented, called when the active display mode changes.

// The widget may wish to change its preferredContentSize to better accommodate the new display mode.

- (void)widgetActiveDisplayModeDidChange:(NCWidgetDisplayMode)activeDisplayMode withMaximumSize:(CGSize)maxSize {
if (activeDisplayMode == NCWidgetDisplayModeCompact) {
       self.preferredContentSize = CGSizeMake(0, 110);
} else {
    self.preferredContentSize = CGSizeMake(0, 170);
   }
}

现在，我们可以看到一个带有展开和折叠功能的widget了。

注意点：

  • 如果初始化的时候将widget设为折叠状态，那么右上角就不会显示折叠/展开按钮
  • 明明设为展开，可是第一次显示还是折叠状态，这是由于在确认widget的mode之前就设置了preferredContentSize的值

数据共享 - App Groups

widget和主app进行数据共享有如下的几种方式：

  • NSUserDefaults
  • NSFileManager

首先我们在开发者后台创建一个具有App Groups的App ID

创建主应用的App ID和widget的App ID

主应用的的App ID和Widget的App ID都需要添加App Groups功能，选择对应的App Groups ID


最后生成对应的Provisioning Profile文件，主应用和widget都需要有对应的。

回到项目工程，在containing App和widget两个target的Capabilities里，有个App Groups选项，打开这个选项，勾选上App Groups ID，要确保下面的三个小对勾都是正常的。
然后我们就可以通过NSUserDefaults和NSFileManager进行数据共享了。

NSUserDefaults
存：
NSUserDefaults *shared = [[NSUserDefaults alloc]initWithSuiteName:@"group.widget.**"];
[shared setBool:@(YES) forKey:kWidgetTestKey];
[shared synchronize];
取：
NSUserDefaults *shared = [[NSUserDefaults alloc]initWithSuiteName:@"group.widget.**"];
BOOL testValue = [shared boolForKey:kWidgetTestKey];

NSFileManager
存：
NSURL *containerURL = [[NSFileManager defaultManager] containerURLForSecurityApplicationGroupIdentifier:@"group.widget.**"]; 
containerURL = [containerURL URLByAppendingPathComponent:@"Library/Caches/widget"]; 
NSString *value = @"my name"; 
BOOL result = [value writeToURL:containerURL atomically:YES encoding:NSUTF8StringEncoding error:nil];
取：
NSURL *containerURL = [[NSFileManager defaultManager] containerURLForSecurityApplicationGroupIdentifier:@"group.widget.**"]; 
containerURL = [containerURL URLByAppendingPathComponent:@"Library/Caches/widget"]; 
NSString *value = [NSString stringWithContentsOfURL:containerURL encoding:
NSUTF8StringEncoding error:nil];

SuiteName和GroupIdentifier就是我们创建的App Groups名称。

共享代码

  • Framework
  • pod引用组件库

关于pod引用我们自己组件库的方式介绍一下。我们的podfile文件一般是如下写法：

platform :ios, '7.0'
source 'git@192.168.1.10:ios/DDSpecRepo.git'

pod 'HTMLAttributedString', '~> 0.0.1'
pod 'HMSegmentedControl', '~> 1.5.2'
pod 'UITableView+FDTemplateLayoutCell', '~> 1.4'
pod 'tingyunApp'
pod 'GrowingIO'
pod 'UICountingLabel'

target 'HostApp' do
end
target 'TestHostApp' do
 pod 'JPFPSStatus', '~> 0.1'
end

如果我们直接加入widget的target，就会引用到多余的第三方库，这是我们可以使用abstract_target关键字，它可以使多个target共享相同的pod。最终写法如下：
​
platform :ios, ‘7.0’
source ‘git@192.168.1.10:ios/DDSpecRepo.git’

abstract_target 'HostApp' do
pod 'HTMLAttributedString', '~> 0.0.1'
pod 'HMSegmentedControl', '~> 1.5.2'
pod 'UITableView+FDTemplateLayoutCell', '~> 1.4'
pod 'tingyunApp'
pod 'GrowingIO'
pod 'UICountingLabel'

target 'HostApp' do
end
target 'TestHostApp' do
 pod 'JPFPSStatus', '~> 0.1'
end
end

abstract_target 'Extension' do
pod 'SDWebImage'

target 'todayWidget' do
end
target 'TestTodayWidget' do
end
end

这样我们的主App就可以使用它自己所需的第三方库，我们的widget也可以只引用自己需要的组件，不会受到主App的影响。
当我们引用一些自己的基础服务组件的时候，需要注意一点，由于widget中是没有UIApplication类的，所以我们无法引用使用了UIApplication的组件或第三方库，会导致编译不过。

网络部分

开始我是想引用主App中的网络请求组件，不过由于牵扯的较多，而且有使用到UIApplication，所以只能放弃。
由于widget所需的数据不多，一般也就一个接口，所以我们直接使用NSURLSession进行网络请求就可以了，然后使用JSON映射的库或者自己使用KVC去生成模型对象。
